# Contributing to MADO

MADOへの参加を歓迎する。コードの変更だけでなく、ドキュメントの修正・Issue整理・導入事例の追加もすべて正式な貢献として扱う。

## MADOの開発スタンス

MADOは北海道芽室町の窓口業務という**具体的な現場**から生まれた。
方向性はメンテナーが現場の実運用をベースに決めている。

ただし、これは閉じた開発という意味ではない。

- 技術的な改善提案は積極的に議論して取り込む
- 他自治体の現場知見は設計に直接反映される可能性がある
- セキュリティ・DB設計などの専門知識は特に歓迎する

**「ガラリと変える」ことはないが、「大きく取り込む」ことはある。**
まずIssueで話しかけてほしい。

---

## フォークについて

各自治体の固有仕様への対応は、**フォークで自由に派生してよい**。

MADOに取り込む基準は「芽室町の窓口でも採用できるか」「小規模自治体に共通するニーズか」。
この基準を超えるカスタマイズはフォークが適切な選択肢になる。

**フォークしても、Issueには参加してほしい。**

フォークで実装した場合でも、設計の発想・迷いどころ・現場の知見はIssueで共有してほしい。
各自治体の現場から生まれた議論が、将来の標準仕様改善の種になる。

> **フォーク例：**  
> queueの「職員が発券するモード」を「来庁者がタッチして自分で発券するモード」に変えたい  
> → フォークで対応。設計上の検討事項はIssueで共有歓迎

## 参加方法の種類

| 種類           | 説明                       | 必要なスキル               |
| ------------ | ------------------------ | -------------------- |
| **バグ報告**     | 動作しない箇所をIssueで報告         | なし                   |
| **ドキュメント修正** | README・手順書の誤字・改善         | なし                   |
| **導入事例追加**   | FORKED_SITES.mdへの自治体情報追加 | なし                   |
| **Issue整理**  | ラベル付け・重複確認               | なし                   |
| **コード修正**    | バグ修正・機能追加のPR             | Python / Flask（queue）、Next.js / TypeScript（hub 以降） |
| **設計レビュー**   | Issueでの設計議論への参加          | ドメイン知識があれば歓迎         |

## ラベル運用

| ラベル | 意味 |
|---|---|
| `good first issue` | 初めての方でも取り組める |
| `help wanted` | メンテナー以外の力を借りたい |
| `bug` | バグ報告 |
| `documentation` | ドキュメントのみの変更 |
| `adoption` | 導入自治体の登録 |
| `discussion` | 設計・方針の議論 |

`github.com/Memuro-Town/MADO-queue/contribute` から `good first issue` 一覧が確認できる。

---

## ブランチ・PRのルール

1. `main` への直接pushは禁止。PRを通すこと
2. ブランチ名にIssue番号を含める（例：`fix/123-printer-connection`）
3. PRの本文に `Fixes #番号` を書くとIssueと自動で連携する
4. 実装中はDraft PRを使って「作業中」であることを示してよい

## AI Assistance

MADOはVibe Codingで開発されており、AIによるコード生成を歓迎する。

PRを送る際は以下を記載すること：
- **使用したAIツール**（例：Claude、GitHub Copilot）
- **利用した範囲**（例：関数全体の生成 / リファクタリングの提案のみ）
- **人間による動作確認の有無**

コードの生成をAIに任せてよいが、**動作確認とロジックの理解の責任は投稿者にある**。

---

## 動作保証・免責

本リポジトリのコードは参考実装である。各自治体での法令適合性・業務適合性は、導入する主体が確認すること。PRのマージは動作保証を意味しない。

本番環境のインシデントは、当該自治体の情報担当部署に連絡すること。

---

## 質問・相談

開発・コントリビューションに関する質問は Issue を使うことを推奨する。なお、各自治体の導入・運用に関する個別サポートには対応できない場合がある（構築支援は委託を想定している）。
